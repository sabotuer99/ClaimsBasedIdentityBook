Web Application with Information Card SignIn

This sample shows how to write an STS that supports username/password backed managed information cards. The username and the password are validated using a custom username security token handler. The STS is hosted using a custom UserNameCardStsHostFactory.

Projects included

1.       CustomUserNameCardStsHostFactory: This project shows how to build an identity provider, which can issue username/password backed managed cards and issue SAML1.1 bearer tokens for those cards. This part is used by the CustomUserNameCardStsHostFactoryWebSite project.

2.       CustomUserNameCardStsHostFactoryWebSite: This project shows how to request a username/password backed managed card and read the claims in the issued token. It is a simplistic example and does not sign the user in.

Steps needed to run the sample

1.       Run the Setup.bat file to create virtual directories.

Running the sample

·         Go to the web site https://localhost/CustomUserNameCardStsHostFactory/Default.aspx to download a managed card and install it. The username is “terry” and password is “123”.

·         Go to the web site https://localhost/CustomUserNameCardStsHostFactoryWebSite/Default.aspx to log in using the managed card.

·         Run the Cleanup.bat file to remove virtual directories.
