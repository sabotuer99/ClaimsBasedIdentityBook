Using Claims in IsInRole()

The purpose of this sample is to demonstrate how to use IIdentity.IsInRole with claims. The sample also shows how claims issued by an Identity Provider may be mapped to claims that the Relying Party understands. This process, called “Claims Transformation,” allows the Relying Party to build services and web pages that understand claims which are valid within the Relying Party’s realm. This process allows the Relying Party to trust relationships with various Identity Providers and manage these relationships with policies managed outside of the code backing individual pages or services.

Projects included in the sample

BookHostingService

Relying Party that allows users to either edit or view a book hosted at this service based on the claims issued by an Identity Provider

BookPublisherSTS

Security token service that authenticates users with Username and Password. Issues a token with claims about whether the user is an Editor or a Reviewer

Steps needed to run the sample

1.       Run the Setup.bat batch file with administrative privileges to setup the sample. This includes creating the following virtual directories in IIS with the specified authentication method(s):

2.       Build the solution.

3.       Browse to https://localhost/BookHostingService/Default.aspx.

4.       The browser will redirect to the BookPublisherSTS. To acquire a token with ‘Editor’ privileges, log in as ‘Bob’. The password for this user is ‘Bob’. To acquire a token with ‘Reviewer’ privileges, log in as ‘Joe’. The password for this user is ‘Joe’.

5.       After you log in, the browser will redirect back to the BookHostingService. If you logged in as Bob, you will be able to edit the contents of the book and save your changes. However, if you log in as Joe, you will only be able to review the contents of the book.

6.       To log out of the session, click the ‘Sign Out’ link at the bottom of the page.

7.       To clean up the virtual directories and certificates created by this sample, run Cleanup.bat from a command prompt with administrative privileges.
