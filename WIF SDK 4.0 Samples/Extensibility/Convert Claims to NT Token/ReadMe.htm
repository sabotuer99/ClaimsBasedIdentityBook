Claims To Windows Token Service (C2WTS)

The purpose of this sample is to show how to use the Windows Token Service (C2WTS) to produce Impersonate-level WindowsIdentity objects from SAML tokens which contain a Kerberos unique principal name (UPN) claim.

To use the C2WTS:

1.       Configure the Microsoft Windows Identity Foundation’s SAML security token handlers to automatically map UPN claims to WindowsClaimsIdentity objects, or

2.       Explicitly use the methods exposed by the S4UClient class to acquire a WindowsIdentity object.

The WindowsIdentity objects produced by using the Windows Token Service allow impersonation. This sample demonstrates how to access a back-end service that prints the user name of the caller after impersonating. Typically the back-end service would perform some operation that required impersonation, such as accessing a local machine resource protected by Windows security permissions.

If the back-end service is not located on the same machine as the service which performs the impersonation, the domain administrator will need to perform further configuration steps in order to allow the impersonating service to use the constrained delegation feature in Active Directory.

Projects included in the sample

AccessService1

A WCF service that uses the automatic mapping feature to produce a WindowsClaimsIdentity from an incoming UPN claim. Once the windows identity is retrieved, impersonation is done to access the back-end print service.

AccessService2

A WCF service that uses the S4UClient.UpnLogon method to access the C2WTS service. The service will log on the user represented by the UPN using the Kerberos S4U feature. Once the new windows identity is retrieved, impersonation is done manually to access the back-end print service.

NOTE: See the comments in the Client project’s Program.cs file to use this access service in lieu of AccessService1.

Client

A simple WCF application that communicates with one of the access service implementations.

Common

Common code base for the other projects.

PrintIdentityService

The print identity service simply prints the current identity’s name that is accessing this service. Note: This service may access resources protected with Windows security on the same machine as if it was executing as the calling identity.

SecurityTokenService

The security token service that issues a UPN claim.

Steps needed to run the sample

1.       Open the solution file.

2.       Open SecurityTokenService\CustomSecurityTokenService.cs and edit the GetOutputClaimsIdentity() method to specify the correct Upn instead of name@yourdomain.com.

3.       Build the solution.

4.       Configure and run the C2WTS service.

a.       Examine the configuration file for the service. By default this file is located at %ProgramFiles%\Windows Identity Foundation\V3.5\c2wtshost.exe.config

b.      Ensure that you are an allowed caller for the service. By default, the service rejects all callers. Add your NT account name or a group you are a member of to the list of allowed callers.

c.       Start the service. You may do this from an administrative command prompt by running:
net start c2wts

d.      The service should start with the following output:

The Claims To Windows Token Service service is starting.

The Claims To Windows Token Service service was started successfully.

5.       Run the projects in the solution. For each of the following projects, you may debug the projects individually in Visual Studio by right clicking the project in the Solution Explorer window and selecting “Start new instance” in the Debug submenu. Alternatively, you may open a command window and run the executable that was built in the project’s build directory. By default, this subdirectory is “bin\Debug.”

a.       AccessService1. The access service will start with the following output:

The access service 1 has started at http://localhost:8080/AccessService1.

Press [Enter] to stop.

b.      PrintIdentityService. The print identity service will start with the following output:

The print identity service has started at http://localhost:8083/PrintIdentityService.

Press [Enter] to stop.

c.       SecurityTokenService. The security token service will start with the following output:

The security token service has started at http://localhost:8081/STS.

Press [Enter] to stop.

d.      Client. Run this project last. The client will start with the following output:

The client has started and is accessing the service hosted at address http://localhost:8080/AccessService1.

Press [Enter] to stop.

While this project is running the print identity service command prompt should print the impersonated user’s name.

6.       When you are done exploring this sample, you may stop the service and edit the C2WTS configuration back to the default.

a.       Stop the service by running the following in an administrative command prompt:
net stop c2wts
