Custom RequestSecurityToken

This sample demonstrates a technique to add custom XML elements to the WS-Trust RequestSecurityToken (RST) messages sent to a SecurityTokenService. In addition, this sample shows how to set a custom request serializer in a Microsoft Windows Identity Foundation SecurityTokenService (STS). There are three core components in this sample: the client, the STS, and the relying party service.

The client uses standard extensibility mechanisms found in the Windows Communication Framework to serialize an additional element in the RST message sent to the STS in order to acquire an issued token.

The STS uses the Windows Identity Foundation to host a WS-Trust token issuer. The sample illustrates how to set a custom request serializer of type CustomWSTrust13RequestSerializer to the SecurityTokenServiceConfiguration object used to configure the STS. This custom serializer adds the custom XML element to the Properties collection on the RequestSecurityToken object provided to it in the ReadXmlElement method. The STS can then take action based on this custom data by consulting the RequestSecurityToken object’s Properties collection. In this sample, it simply verifies that such an element was placed in the Properties collection in the ValidateRequest method of CustomSecurityTokenService.

The relying party is a simple WCF web service.

Projects included in the sample

CustomRequestSecurityToken

This project contains the client, STS, and relying party service.

Steps needed to run the sample

1.       Open the solution file.

2.       Build the solution.

3.       Run the sample (F5). You should see the following output on the command prompt:

The echo service has started at http://localhost:8080/EchoService.

The security token service has started at http://localhost:8081/STS.

The client sent a request to the STS to retrieve a SAML token and then sent the hello request to the echo service.

The STS received the custom element.

The echo service finally returned 'Hello'.

Press [Enter] to continue.
