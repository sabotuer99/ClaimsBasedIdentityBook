Claims Aware AJAX Application

This sample demonstrates how to use the Windows Identity Foundation with an AJAX application.

Projects included in the sample

AjaxRP

An AJAX web application.

AjaxSTS

A Security Token Service (STS) that issues claims used by the Relying Party (RP).

Steps needed to run the sample

1.       Run the Setup.bat batch file with administrative privileges to setup the sample. This includes creating the following virtual directories in IIS with the specified authentication method(s):

Virtual Directory

Folder Name

Enabled Authentication

AjaxSample_RP

AjaxRP

Anonymous

AjaxSample_STS

AjaxSTS

Anonymous, Forms

2.       When you are done exploring this sample, run the Cleanup.bat batch file with administrative privileges to delete the virtual directories and the database created in step 1.

Running the sample

·         Browse https://localhost/AjaxSample_RP/Default.aspx. This page will redirect your browser to the passive federation endpoint at https://localhost/AjaxSample_STS/Default.aspx.

·         You will be asked for a username and password. Use “JohnDoe” as the username and “JohnDoe123” as the password.

·         The STS will issue you a security token and redirect you to the RP located at https://localhost/AjaxSample_RP/Default.aspx. Note that the user name is displayed on the page.

·         Click on the “Call service” button. The sample will time out the cookie containing the token issued by the STS after ten seconds.

·         After ten seconds, click “Call service” again. Since the cookie expired, you will be presented with the Sign In page.

Security Note

Note that the mode of storing user credentials for forms-authentication in web.config is used in this sample for illustrative purposes only. In a real deployment, consider using a membership provider to store and validate user credentials.

Note

The relying party application(s) in this sample can accept tokens issued by an Active Directory Federation Services 2.0 STS instance instead of the sample STS instance. You would need to modify the STS configuration parameters such as ‘issuer’ in the application’s configuration. Refer to the AD FS 2.0 product documentation for instructions on how to register a relying party application with an AD FS 2.0 STS instance.
