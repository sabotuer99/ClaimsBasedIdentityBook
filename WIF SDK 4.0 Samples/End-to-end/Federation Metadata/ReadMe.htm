Federation Metadata Scenario

This sample demonstrates how to dynamically generate federation metadata for a Security Token Service.

 

The Windows Identity Foundation ships with a Visual Studio template for creating a WCF Security Token Service. This template includes a static file, FederationMetadata.xml, which enables the token service to establish a federated trust relationship with a claims aware web service. In practice, however, it may be necessary for attributes of the service to change over time and it is better to generate the metadata dynamically to reflect the actual state of the token service.

 

Projects included

1.       ClaimsAwareService: A WCF service based on the Windows Identity Foundation template for a web-hosted claims-aware WCF service. The service provides a contract, IQuote, for obtaining a quotation for authenticated users.

2.       STSService: Service that can federate with ClaimsAwareService to provide authentication tokens.

3.       ClientWebSite: A site which will display the quote generated by the service.

2.        Open the solution file.

3.        Build projects in the solution.

4.        Launch the client site: http://localhost/clientwebsite/default.aspx . Text will display indicating the user has no quote claims.

5.        Right-click on the ClaimsAwareService and select Add STS Reference...

·         Remove the port number that will be in the application URI. For example, change http://localhost:55231/ClaimsAwareService/QuoteService.svc to http://localhost/ClaimsAwareService/QuoteService.svc.

·         Accept the service and contract names on the next screen.

·         On the following screen, select the “Use an existing STS” radio button. Enter http://localhost/stsservice/federationmetadata/2007-06/federationmetadata.ashx as your STS federation metadata location. This endpoint will generate new metadata dynamically, so the claims offered by the STS can be changed.

Optionally, click “Test location…” to verify that this endpoint is functional. A browser window should display a federation metadata document. Move on to the next screen by selecting the “Next” button.

·         On the following screen, select the “Enable encryption” radio button. Choose the “Select an existing certificate from store” radio button and click the “Select Certificate…” button to display the operating system’s certificate selection dialog. Choose the localhost certificate. After you dismiss the certificate selection dialog, verify that the certificate selected is “CN=localhost”.

The next screen will display the offered claims. Move on to the next screen by selecting the “Next” button.

·         The next screen will summarize all changes the tool will make.

6.        Open Web.config in the ClaimsAwareService project and uncomment the following line:
<add claimType="http://schemas.microsoft.com/WindowsIdentityFramework/Samples/ClaimTypes/QuotationClass" isOptional="true" />
This is a new claim advertised by the federation metadata.

7.        Open the Web.config in the ClientWebSite project. Remove or comment out the <configuration>/<system.serviceModel>/<client>/<endpoint> node. If you do not do this, the following step will create multiple endpoint nodes for the same “QuoteService.IQuote” contract.

8.        Update the QuoteService service reference created in step 3 by opening the App_WebReferences folder and selecting “Update Web/Service References” from the context menu for this node. Save any changes made to the web.config file.

9.        Launch the client site: http://localhost/clientwebsite/default.aspx again. Text will display a quote based on the extra claims supplied.

Note

The relying party application(s) in this sample can accept tokens issued by an Active Directory Federation Services 2.0 STS instance instead of the sample STS instance. You would need to modify the STS configuration parameters such as ‘issuer’ in the application’s configuration. Refer to the AD FS 2.0 product documentation for instructions on how to register a relying party application with an AD FS 2.0 STS instance.
