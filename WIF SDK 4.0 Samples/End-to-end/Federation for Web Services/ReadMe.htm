Federation Active Scenario

This sample shows a typical federation scenario with a book store client, a home realm STS (which is in the same domain as the client), a book store STS, and a book store service.  The home realm STS is the identity provider (IP) STS, the book store STS is the relying party (RP) STS, and the book store service is the RP.

 

 

This sample is similar to the federation sample in the Windows Communication Foundation (WCF) SDK but uses Microsoft Windows Identity Foundation to implement the STSes. Comparing the two samples, one can see it is a simpler task to write an STS with Microsoft Windows Identity Foundation.

 

Projects included

1.       BookStoreClient: Client application built on WCF.

2.       BookStoreService: Service that hosts the bookstore functionality.

3.       BookStoreSTS: Security token service that BookStoreService relies on.

4.       HomeRealmSTS: The client's IP STS.

5.       ScenarioCommon: Scenario specific constants and an X509 certificate helper.

2.       Open the solution file. Build projects in the solution, and launch the solution.

4.       The BookStoreClient window will open.

5.       Click the “Browse Books” button to browse the book list.

6.       Select a book and click the “Buy Book” button to purchase a book.  Note that if the book “Book Title One” is chosen the default purchase limit is exceeded and the bookstore service returns a fault.

7.       Run the Cleanup.bat file to remove the sample.

Note

The relying party application(s) in this sample can accept tokens issued by an Active Directory Federation Services 2.0 STS instance instead of the sample STS instances. You would need to modify the STS configuration parameters such as ‘issuer’ in the application’s configuration. Any of the STSs in this sample can be replaced by an AD FS 2.0 STS. You must ensure that the AD FS 2.0 STS configuration parameters are reflected in the remaining STS in the sample, as appropriate. Refer to the AD FS 2.0 product documentation for instructions on how to register a claims provider and a relying party application with an AD FS 2.0 STS instance.
